# 存序 — 产品介绍与实现纲要

**定位**：你的个人 **生活技巧与注意点库**。把与健康、下厨、家务、出行、旅行、花钱、就餐、家居、娱乐相关的 **可复用经验、踩坑记录、事前检查项** 结构化保存；在真正做事时 **快速调出、按条对照**，减少因遗忘、信息差或突发失误带来的心态波动，做到 **心中有数、节奏可控**。

**不做**：医疗/法律等专业领域的「结论替代」——产品只存 **你的笔记与公开常识整理**，重要决策仍需咨询专业人士。

---

## 1. 产品怎么设计，能帮助快速知晓技巧与注意点

### 1.1 核心目标

- **按生活场景分区**：九大场景与页签一一对应（见下文 `TABS`），进入某 Tab 即只看该场景下的条目，减少噪音。
- **快查快用**：支持 **标题搜索、标签筛选**；单条支持 **摘要 + 正文**（清单、步骤、禁忌可写在正文）；重要条目可 **置顶** 或标 **常用**。
- **降低「临场翻车」**：对适合的场景提供 **检查清单（Checklist）** 形态（如旅行出发前、长途开车前），勾选完成即心理有底。
- **沉淀踩坑**：专门鼓励记录 **「易错点」「曾经翻车的条件」**，与成功技巧并列，形成正反两面笔记，缓解信息差带来的不确定感。

### 1.2 与九类场景对应

| 场景（页签） | 典型内容示例 |
|--------------|--------------|
| 健康 | 作息、运动频率、常见不适的应对备忘（非诊疗结论）、体检周期提醒等。 |
| 做饭 | 火候、预处理、调味比例、失败案例与改进。 |
| 家务 | 清洁顺序、工具与耗材、周期维护（换滤芯等）。 |
| 开车 | 路况习惯、保养周期、长途/雨雪注意点、保险与证件备忘。 |
| 旅行 | 打包清单、目的地贴士、签证/票务时间节点。 |
| 消费 | 比价习惯、退换规则备忘、订阅与账单检查节奏。 |
| 吃饭 | 外食选择原则、过敏与忌口、聚餐礼仪或预算习惯（与「做饭」区分：偏「怎么吃/去哪吃」）。 |
| 装修 | 量房与验收、工序顺序、环保与合同注意点。 |
| 娱乐 | 观影/游戏/活动的时间与预算边界、设备与账号备忘。 |

## 2. 前端页面怎么设计

### 2.1 页签常量（与 `chat/page.tsx` 一致的模式）

与「晤言」相同，使用 **`role="tablist"`**、`role="tab"`、`role="tabpanel"`，并配置 **`aria-selected`、`aria-controls`、`tabIndex`** 等，保证键盘操作与读屏可读。

**九个页签**建议写死为常量（顺序即产品默认顺序）：

```text
const TABS = [
  "健康",
  "做饭",
  "家务",
  "开车",
  "旅行",
  "消费",
  "吃饭",
  "装修",
  "娱乐",
] as const
```

- **当前选中 Tab** 对应后端查询参数 **`category`**（或 `scene`）的枚举值，建议与英文常量映射（如 `HEALTH`、`COOKING`…）便于接口与数据库一致。
- 切换 Tab 时 **刷新列表**；可将 `category` 同步到 **URL 查询参数**（如 `?tab=travel`），便于收藏与分享单场景视图。

### 2.2 单 Tab 内布局（每个场景页结构一致）

1. **顶栏**：当前场景标题（可与 Tab 文案一致）、**搜索框**（搜标题/标签/正文关键词）、可选 **仅看常用** 开关。  
2. **列表区**：卡片或表格列表；每条展示 **标题、摘要一行、标签、更新时间**；支持 **加载中 / 空状态**（该场景下暂无笔记时的引导文案）。  
3. **分页**：条目较多时分页或虚拟列表（按团队规范选型）。  
4. **详情**：侧栏或全屏/对话框打开；展示 **正文**（Markdown 渲染）、**清单勾选**（若该条为 checklist 类型）、**编辑 / 删除**。  
5. **新建**：浮动按钮或顶栏「新建」；**默认 category = 当前 Tab**，减少选错场景。

### 2.3 无障碍与体验

- 表单与搜索框带 **可见 `label` 或 `aria-label`**。  
- 清单勾选使用 **语义化 `checkbox`** 与可聚焦样式（`focus-visible`）。  
- 异步加载列表使用 **`aria-live="polite"`** 简短提示（可选）。

## 3. 表怎么设计 — 需要哪些表

表名可加前缀（如 `cx_` 表示存序），以下为逻辑模型。

### 3.1 核心条目

| 表名（建议） | 用途 |
|--------------|------|
| `life_tip` | 主表：`id`，`title`，`summary`（短摘要，列表展示），`body`（Markdown 长文），`category`（枚举，对应九大场景之一），`is_pinned`（置顶），`is_favorite`（常用），`created_at`，`updated_at`，`owner_id`（多用户时）。 |
| `life_tip_tag` | 标签：`name`（唯一归一化）。 |
| `life_tip_tag_rel` | 多对多：`tip_id`，`tag_id`。 |

### 3.2 清单（可选，第一版可合并进正文）

若希望 **勾选状态可持久化、跨设备同步**：

| 表名（建议） | 用途 |
|--------------|------|
| `life_checklist_item` | `tip_id`，`sort_order`，`label`，`checked`（布尔），`updated_at`。 |

若第一版追求简单，可用 **Markdown 任务列表** 存在 `body` 中，由前端解析勾选（不持久化或仅存整段 Markdown）——产品迭代时可再拆表。

### 3.3 附件（可选）

| 表名（建议） | 用途 |
|--------------|------|
| `life_tip_attachment` | 图片/链接：`tip_id`，`url` 或存储键，用于小票、示意图等。 |

## 4. 后端接口需要怎么设计

**约定**：统一前缀如 `/codelab/cunxu`（与现有 `/codelab` context-path 一致）；响应结构可与现有 `CommonResponse` 对齐。

### 4.1 条目 CRUD

- `GET /tips` — 分页；**必填或强约束**：`category`（九大枚举之一），与当前 Tab 一致；可选：`q`（关键词）、`tag`、`favoriteOnly`、`pinnedFirst`。  
- `GET /tips/{id}` — 详情（含标签、可选 checklist、附件列表）。  
- `POST /tips` — 新建；`category` 由前端传入，服务端校验属于九类之一。  
- `PUT /tips/{id}` — 更新。  
- `DELETE /tips/{id}` — 删除或软删。  

### 4.2 标签

- `GET /tags` — 标签列表（可按 `category` 过滤使用频率，可选）。  
- 标签可在新建/更新条目的请求体中 **一并提交**（数组），由服务端 **幂等** 维护 `life_tip_tag_rel`。

### 4.3 清单行（若独立建表）

- `PUT /tips/{id}/checklist` — 批量更新勾选状态与排序。  
- 或 `PATCH /tips/{id}/checklist/items/{itemId}` — 单条勾选。

### 4.4 附件（若启用）

- `POST /tips/{id}/attachments` — 上传。  
- `DELETE /attachments/{attachmentId}` — 删除。

---

## 5. 小结

**存序** = **九大生活场景页签** + **每场景下的技巧/注意点条目** + **搜索与标签** + **（可选）清单与附件**；目标是让你在做事前 **快速对齐已知经验**，减少失误与心态波动。

前端采用与 **`chat/page.tsx` 相同的 `TABS` + `tablist` 页签模式**，共 **9 个 Tab**；后端以 **`life_tip.category` 枚举** 与列表查询为核心即可落地第一版。
