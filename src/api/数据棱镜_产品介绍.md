# 数据棱镜 — 产品介绍与实现纲要

**定位**：围绕你的「20% 短线 + 80% 长线」「多市场、多品种、抄底偏好、分批买入 + 极端梭哈」逻辑，提供**指标看板**集中展示行情与衍生指标；通过**告警（信号）**在关键时刻提示机会与风险。**不做交易流水、不做规则配置页、不做数据质量独立模块**——触发阈值在**后端/计算任务代码中写死**（与本文「附录：写死规则常量」一致），版本迭代靠发版而非界面改参。

---

## 附录：写死规则常量（产品侧约定，实现时以代码为准）

| 规则 | 写死内容（示例，可按发版调整） |
|------|--------------------------------|
| 日常抄底 | 纳指**单日跌幅 ≥ 2%** 记一条信号；监控列表内个股**单日跌幅 ≥ 6%** 记一条信号；文案可带建议单笔区间 **1000–2000 USD**（仅提示，不落账）。 |
| 梭哈 | **标普**自锚定高点**最大累计跌幅 ≥ 20%**、**纳指 ≥ 25%** 各记高优先级信号。 |
| 锚定回撤算法 | 由实现选定一种并写死在代码中（如滚动窗口最高价或阶段高点），前后端/批任务共用同一实现。 |

## 1. 产品怎么设计，能更好支撑交易

### 1.1 核心目标

- **走势看板**：一屏查看**全部监控品种**的**价格走势线**，通过**统一下拉框**切换 **一周内 / 1 年内 / 3 年内**。  
- **指标看板**：把多市场关键标的与指数的**价格位置、日涨跌、距阈值距离、分市场/分桶占比**等放在同一屏（另一 Tab），减少切 App、切表。
- **告警（信号）**：规则触发即产生**信号记录**；你在**页面右上角铃铛**中查看**最近 3 天内**出现的信号，无需单独进「流水」或「待办列表页」也能看到近期提醒。
- **写死规则、减少干扰**：不在产品内提供「改阈值」入口，避免临盘随意改规则；调整策略通过改代码与发版完成。

### 1.2 功能模块（与交易逻辑对应）

| 你的逻辑 | 产品能力 |
|----------|----------|
| 20% 短线 / 80% 长线；A 股 AI 相关 ETF / 港股 / 美股指数与个股 | **走势看板（纯线图）**：同一页展示**监控清单内全部品种**的价格走势（折线/收盘线），周期由页顶**统一下拉框**切换。**指标看板**：分市场、分主题标签展示当日指标、回撤与表格摘要。 |
| 抄底偏好、不喜追高 | 看板可对「单日大涨」等做**浅色提示**（阈值写死），与「大跌触发信号」区分。 |
| 长线布局与 2026 年节奏 | **标的与仓位 Tab**：展示标的池、标签、持仓快照（若接入）、是否在长线关注清单；**不写流水、不对账成交**。 |
| 纳指 -2% / 个股 -6% / 标普 -20% / 纳指 -25% | **信号**：定时或盘后计算命中即写入 `signal`；**铃铛**拉取近 3 日记录展示标题、时间、类型、关联标的/指数摘要。 |

### 1.3 产品原则

- **数据棱镜是决策辅助，不是自动下单**；信号仅为提醒，是否交易由你自行执行。
- **无交易流水模块**：本产品不记录买卖明细、不提供对账与频次统计（若需要可另做独立应用）。

## 2. 前端页面怎么设计

### 2.1 全局：告警铃铛（右上角）

- 在数据棱镜布局**右上角**固定 **铃铛图标**；可带未读数量角标（可选：近 3 日内未点开过的信号数，需后端字段或本地已读态）。
- 点击展开 **Dropdown / Popover**：列表展示 **最近 3 个自然日（或 72 小时，实现二选一写死）** 内产生的信号，按时间倒序。
- 每条信号：**类型**（如「纳指日跌幅触发」「个股深跌触发」「标普回撤梭哈阈值临近/已触发」等）、**关联标的或指数**、**触发时间**、**简短说明**（可含写死的建议金额文案）。
- **无障碍**：按钮 `aria-label` 如「告警，近三日信号」；列表可用 `role="list"` / 合理焦点管理。

### 2.2 信息架构：页签（与 `chat/page.tsx` 的 `tablist` 一致）

建议使用 **`role="tablist"`** 的页签，例如：

```text
const TABS = ["走势看板", "指标看板", "标的与仓位"] as const
```

说明：**不再包含**「交易流水」「规则与参数」「数据质量」等 Tab；**信号**以铃铛为主入口；**指标看板**内可保留**可选**的「今日信号摘要」小块（与铃铛数据同源）。

1. **走势看板（纯看板 · 全品种线图）**  
   - **唯一主控件**：页面顶部（Tab 内容区顶栏）放置**统一下拉框**（`Select`），选项**写死为三档**（枚举与文案固定）：  
     - **一周内**  
     - **1 年内**  
     - **3 年内**  
   - **展示内容**：监控范围内的**每一个品种**各一张**价格折线（或收盘连线）**小图，按网格排列（如响应式多列）；纵轴为价格（或归一化涨跌幅，二选一在实现中写死并在图角标注），横轴为时间，随下拉框切换**整体**时间窗与拉数区间。  
   - **交互**：下拉变更后**全页图表统一重载**同一套 `range` 参数，避免每图单独选周期。加载中、单品种无数据时该格显示空态或「暂无 K 线」。  
   - **数据粒度（建议写死）**：一周内可用**日 K** 或更细粒度（若数据源允许）；**1 年内 / 3 年内**默认 **日 K**，保证性能与缓存策略一致。  
   - **无障碍**：下拉框带 `aria-label`（如「走势图时间范围」）；图表容器用简要文字说明当前选中范围。

2. **指标看板**  
   - 卡片/栅格：**标普、纳指** 的当日涨跌、自高点回撤%、距写死梭哈阈值的空间（进度或文案）。  
   - 监控列表表格：标的、市场、标签、日涨跌、是否今日已触发信号（图标或标签）。  
   - 可选：短线/长线目标占比 vs 当前持仓占比（数据来自持仓接口，若无持仓则仅展示目标模板文案）。

3. **标的与仓位**  
   - 表格：标的、市场、主题标签、持仓数量/市值（若有）、成本与浮动（若有）；支持按市场、标签筛选。  
   - **长线关注清单**可与标的数据合并展示（字段或标签区分），**不与流水、成交频次联动**。

### 2.3 交互与可视化

- 表格：加载中、空态、分页、搜索（与现有前端规范一致）。  
- **走势看板**：多图布局注意**首屏性能**（虚拟列表、按需请求、或后端一次返回聚合序列）；同页多系列时图例尽量显示代码简称。  
- **指标看板**关键数字：**大数字 + 迷你趋势或进度条**（数据源允许时）。  
- **时区**：各市场字段旁标注数据源日历或时区说明；线图横轴按**各市场交易日**对齐时需在接口或文档中约定规则。

## 3. 表怎么设计 — 需要哪些表

以下为逻辑模型（表名可加前缀如 `dp_`）。

### 3.1 基础

| 表名（建议） | 用途 |
|--------------|------|
| `instrument` | 标的主数据：代码、名称、市场（CN/HK/US）、类型（ETF/股票/指数）、币种、主题标签（关联表或 JSON）。 |
| `instrument_tag` / `instrument_tag_rel` | 标签：如 AI-电力、芯片、半导体、光伏、机器人、黄金等。 |

### 3.2 持仓（无流水）

| 表名（建议） | 用途 |
|--------------|------|
| `position` | **当前持仓快照**（可选）：标的、数量、成本、市值、策略桶；由导入或外部同步更新，**不由此产品记流水**。 |

### 3.3 行情与回撤状态

| 表名（建议） | 用途 |
|--------------|------|
| `market_quote_daily` | 日 K：标的/指数、交易日期、收盘、涨跌等。 |
| `index_drawdown_state` | 标普/纳指：锚定高点、当前回撤、最大回撤、是否已触发梭哈类信号（按写死规则更新）。 |

### 3.4 信号（告警数据源）

| 表名（建议） | 用途 |
|--------------|------|
| `signal` | 每次规则命中一条：类型枚举、触发时间、关联 instrument 或指数代码、说明文案、结构化快照 JSON（当日跌幅、阈值等）。**无「关联成交」外键**。 |

### 3.5 长线关注（可选）

| 表名（建议） | 用途 |
|--------------|------|
| `long_term_watchlist` | 标的级别「持续买入关注」与备注、排序；**不与成交统计挂钩**。 |

### 3.6 已砍掉的部分（明确不写）

- ~~`trade_ledger`~~、~~流水相关表~~。  
- ~~`trading_rule`~~、~~规则版本表~~（规则用代码常量 + 单元测试约束）。

## 4. Spring Boot Java 后端接口怎么设计

**约定**：统一前缀如 `/codelab/dataprism`；响应结构与现有 `CommonResponse` 对齐。

### 4.1 标的与标签

- `GET /instruments` — 分页/筛选。  
- `GET /instruments/{id}` — 详情。  
- `POST/PUT /instruments`（按需）— 维护标的与标签。

### 4.2 持仓（无流水）

- `GET /positions` — 当前持仓列表（无则空列表）。  
- `POST /positions/sync`（可选）— 外部快照导入。

### 4.3 走势图数据（走势看板专用）

- `GET /charts/series` — 查询参数：**`range`** = `1w` \| `1y` \| `3y`（与前端三档下拉一一对应，写死枚举）。  
  - 返回体建议：`{ range, asOf, series: [ { instrumentId, symbol, name, market, points: [ { t, close } ] } ] }`；`points` 为时间升序，**一次返回当前监控列表内全部品种**（或分页版本 `GET /charts/series?range=1y&page=&pageSize=`，由产品决定；纯看板为体验优先可优先「单次全量 + 缓存」）。  
- 实现上基于 `market_quote_daily`（及更细表若未来有）按 `range` 截断起止日期；**禁止**由前端对每个标的各打 N 次请求（除非做懒加载优化并有明确规范）。

### 4.4 指标看板聚合（推荐 BFF）

- `GET /dashboard/metrics` — 指标看板用：指数回撤、距阈值、监控标的当日涨跌摘要、分市场占比等（内部多表组装）。  
- `GET /dashboard/drawdown` — 若需单独拆出：标普/纳指回撤与锚定信息。

### 4.5 信号（铃铛与可选指标看板摘要）

- `GET /signals/recent?days=3` — **铃铛专用**：默认 `days=3`，返回时间倒序列表；字段满足前端铃铛与摘要展示。  
- `GET /signals`（可选）— 分页历史，仅供排障或未来扩展，**不作为必选产品功能**。

> **无** `PUT /rules`、**无** 流水类 `GET/POST /trades`。

### 4.6 行情写入（内部）

- `GET /quotes/daily` — 按标的 + 日期区间查询。  
- `POST /quotes/ingest` — 供 Python 或定时任务写入日 K。

### 4.7 定时任务（Java）

- 行情就绪后：按**写死常量**计算是否触发 → 插入 `signal` → 更新 `index_drawdown_state`。  
- 规则逻辑建议集中在单一 Service + 常量类，便于发版变更与测试。

## 5. 哪些适合用 Python 微服务

- **行情采集与清洗**：拉取 A/港/美日 K，调用 Java `POST /quotes/ingest`。  
- **可选**：复杂指标预计算后以 JSON 推给 Java，由 Java **唯一写入** `signal` / `index_drawdown_state`。  
- **可选通知**：企业微信/Telegram 与页面铃铛互补；业务仍以 Java 库表为准。

---

## 6. 小结

**数据棱镜（当前范围）** = **走势看板（全品种线图 + 周期下拉：一周内 / 1 年内 / 3 年内）** + **指标看板** + **右上角铃铛（近 3 日信号）** + **标的与仓位（+ 可选持仓快照）**；**规则在代码中写死**；**不包含交易流水、规则配置页、数据质量 Tab**。

前端：**`TABS = ["走势看板", "指标看板", "标的与仓位"]` + 全局铃铛**；后端：**Spring Boot 管库表、信号落库与 `GET /charts/series`**，Python 可选做行情与计算外包。
